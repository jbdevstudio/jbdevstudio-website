Red Hat JBoss Developer Studio 7.x - Scripted Installation

Scripted Installation


There are two ways to script a JBoss Developer Studio installation using the installer jar.

Bootstrap Method: Run & Repeat


  1. Run the installer using the izpack GUI (with java -jar jbdevstudio-*.jar, then look for a file in the installation folder called InstallConfigRecord.xml.

  2. For subsequent installs, you can then run headlessly with java -jar jbdevstudio-*.jar InstallConfigRecord.xml to repeat the same install config on similar machines. Note that each InstallConfigRecord.xml is machine-specific, using your specific JDK and install folder paths, so you will need to manually adjust the file to use it for different OS platforms, JDKs or paths.
- OR -

Shell Script Method: Run Anywhere


  1. Run a shell script that will generate a suitable jbdevstudio-*.install.xml file for your specific machine's configuration, then perform the installation. Unlike the above aproach, you don't have to install JBoss Developer Studio using the izpack GUI to create the config file.

  2. You can optionally provide a specific JDK to the script, or let it use the default one in the script (c:\progra~1\java\jre7\bin\java for Windows or /usr/bin/java for Linux/OSX).

    • jbds-install.cmd (Windows)
    • jbds-install.sh (Linux or OSX)
